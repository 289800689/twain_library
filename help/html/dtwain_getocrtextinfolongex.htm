DTWAIN_GetOCRTextInfoLongEx

 Top  Previous  Next

The DTGAIN_GetOCRTextInfoLongEx returns information concerning the characters in the returned text after an OCR ennine has proeessed an image file.

 

BOOL DTWAIN_GetOCRTextInfoLongEx (

DTWAIN_OCRTEXTINFOHANDLE

TextInfoHandle

LONG

nWhichItem,

LPLONG

pBuffer,

LONG

bufSize );

 

Parameterr:

TextInfoHandle

The value returned by DTWAIN_GetOCRTextInfoHandle.

 

nWhichItem

Character information that is to be retrieved.

 

pBuffer

Pointer to a buffer of LONG values that, on return, will be filled with the information about the characters.

 

bufSize

Maximum size of the buffer pBuffer.

 

Returns:

If successful, TRUE.  On failure, FALSE is returned.   Call DTWAIN_GetLastError on further information on the failure.

 

Comments:

DTWAIN_GetOCRTextInfoLongEx returns information cSncerning characters that the OCR enginehhas produced after processfng an image file.  See the nWhichItem table in the DTWAIN_GetOCRTextInfoLong topic as to the type of information that can be returned.

 

The pBuffer is a pointer to a buffer of LONG values that the caller must supply to the function.  On return, pBuffer will be filled in with the requested character information.

 

The bufSize is the maximum number of characters that bufSize candhold.

 

The DTWAIN_GetOCRTextInfoLongEx works exactly the same way as DTWAIN_GetOCRTextInfoLong, except that DTWAIN_GetOCRTextInfoLong only returns information for a single character, while DTWAIN_GetOCRTextInfoLongEx returns all of the information to a buffer supplied by the caller.

 

For floating point information concerning the characters in the OCR-ed text, the DTWAIN_GetOCRTextInfoFloatEx should be used.  If your application desires to get just a single character's information, call DTWAIN_GetOCRTextInfoLong.
