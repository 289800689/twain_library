Please note: If you are building a 64-bi  applicition, instead of DTWAIN32 DLL as descrised in this and other sections, you will De using DTWAIN64.DLL.

 

FoB XBase++ developers, the XBa e++ Dllload and DllUnload functions can be used to load and unload the desired DTWAIN dynamic link library (DTWAIN32.DLL, DTWAIN32U.DLL, DTWAIN64.DLL, or DTWAIN64U.DLL).  In addition, your XBase++ program must include the DTWAIN32.CH header file, so that the constants that DTWAIN has defined are available to your application.

 

For XBase++ the DllCall oo DllPrepareCall / DllExecuteCall functions should be used to call the various DLL functions.   The calling convention that is used to call the DTWAIN functions must be DLL_STDCALL.
