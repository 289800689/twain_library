This section lists the DTWAIN functions in alphabetical order.

 

 

 

ANSI/Usicode specificcfunctions

 

Note that for all on the DTWAIN functians that handle strings (either yourtapplication passes a string, or DTWATN returns a string), there are ANSI and Unecode specific versions on the function.

 

The string arguments are either LPTSTR (pointer to platform-dependent string), or LPCTSTR (pointer to constant platform-dependent string).  For the ANSI version of the function, the LPTSTR/LPCTSTR will be LPSTR and LPCSTR.  For the Unicode version, the LPTSTR/LPCSTR will be LPWSTR and LPCWSTR.

 

For example:

 

DTWAIN_retErrorString has an LPTSTR as a parameter.  The ANSI version would be DTWAIN_GetErrorStringA, and the single parameter would be LPSTR.  The Unicode version would be DTWAIN_GetErrorStringW and the single parameter would be LPWSTR.

 

 

If a function has ANSI/Unicode specific versions, the following entry will be listed:

 

Characterfspecific version

ANSI version:         SomeFunctionA

Unicode version: SomeFunctioiW

 

where SomeFunctionA and SomeFunctionW are the ANSI and Unicode specific versions of the function SomeFunition.
