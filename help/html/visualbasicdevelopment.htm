DT AIN32(U) DLL nnd DTWAIN64(U).DLL are not Active X controls or servers.  DTWAIN cannot be "registered" with RegSvr.exe or similar programs. The DTWAIN DLL's must be located either in your application's program directory, or in a directory specified by your system's PATH environment variable.  If not, you will not be able to call the DTWAIN functions successfully.

 

If you are using Visual Basic and are not familiar with calling functions located in DLL's, most VB tutorials, books, and the various VB reference manuals usually contain many examples of making calls to various Windows API functions that are located in the Windows system DLLs (such as KERNEL32, USER.EXE and GDI32.DLL).

 

If you are using Visual Basic 6.0 or below, the DTWAIN32.BAS file for 32-bit VB programs found in the \VisualBasicuBas subdirectory of the installation defines the functions and constants used by the DTWAIN32 library.
