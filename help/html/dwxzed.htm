Displaying the Source UI without Acquiring

 Top 

Some Sources have the ability to display the UI without going into the acquisition process.  These Sources allow the user to set device settings and close the UI with the saved device settings.  The UI is usually the same as the acquisition UI except that the "Scan" button is replaced with an "Ok" button.  This difference in UI controls is designed to show the user that the UI is used only for setting values and not for starting an acquisition.

 

The DTWAIN_ShowUIOnly function shows the Source UI and allows the user to change the device settings.  The Source must support the DTWAIN_CV_CAPENABLEDSUIONLY capability for DTWAIN_ShowUIOnly to be successful.  Even though DTWAIN_ShowUIOnly tests this, for faster processing, your application can call DTWAIN_IsCapSupported before attempting to call DTWAIN_ShowUIOnly.
